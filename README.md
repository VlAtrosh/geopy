# 📁 Geopy Mini-Project: Геокодирование адресов с кэшированием

Geopy Mini-Project — это демонстрационный проект, показывающий возможности библиотеки geopy для геокодирования и обратного геокодирования на примере преобразования адресов в координаты. Вместо ручной обработки API вы получаете автоматическое геокодирование, контроль качества результата, кэширование запросов и расчет расстояний до заданной точки.

Проект создан для наглядной демонстрации возможностей geopy: геокодирование через Nominatim, Rate Limiter для соблюдения лимитов сервиса, кэширование результатов, обработка ошибок и интеграция в ETL-пайплайн.

## 💡 Моё мнение о библиотеке geopy

Geopy — это стандартный инструмент для работы с геоданными в Python. Вместо того чтобы разбираться с API разных геокодеров и писать обработку ошибок вручную, вы просто подключаете нужный сервис и получаете готовые функции для геокодирования и расчета расстояний.

Библиотека проста в использовании — достаточно создать экземпляр геокодера и вызвать .geocode(). Под капотом — адаптеры для Nominatim, Google Maps, Yandex и других сервисов, которые унифицируют работу с разными API. RateLimiter автоматически контролирует частоту запросов, а geodesic позволяет точно рассчитывать расстояния с учетом кривизны Земли.

Для инженерных задач geopy незаменим. Он помогает превращать "сырые" адреса в координаты, обогащать данные для аналитики, проверять географические гипотезы и легко интегрируется с pandas, Jupyter и любыми ETL-процессами.

## Установка

1.Клонируйте репозиторий:

```

```

2.Cоздайте виртуальное окружение (рекомендуется):

```
# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux/macOS
python -m venv .venv
source .venv/bin/activate
```

3. Установите зависимости:

```

```

## 📋 Справка по командам
```
# Базовый запуск (использует input.csv и output.csv)
python main.py

# Указание входного и выходного файлов
python main.py --input my_addresses.csv --output results.csv

# Смена точки интереса (по умолчанию Красная площадь)
python main.py --poi-lat 59.9343 --poi-lon 30.3351

# Указание User-Agent для Nominatim (рекомендуется!)
python main.py --user-agent "my_project/1.0 (contact@example.com)"

# Настройка задержки между запросами (по умолчанию 1 сек)
python main.py --min-delay-seconds 0.5
```

## 🧠 Ключевые концепции

1. Геокодирование через Nominatim
```
from geopy.geocoders import Nominatim

geolocator = Nominatim(user_agent="my_app")
location = geolocator.geocode("Красная площадь, Москва")
print(location.latitude, location.longitude)  # 55.7537 37.6199
```
2. Rate Limiter (ограничение частоты запросов)
```
from geopy.extra.rate_limiter import RateLimiter

geolocator = Nominatim(user_agent="my_app")
geocode = RateLimiter(geolocator.geocode, min_delay_seconds=1.0)

for address in addresses:
    location = geocode(address)  # автоматическая задержка
```
3. Расчет расстояний (геодезические расстояния)
```
from geopy.distance import geodesic

poi = (55.7558, 37.6173)  # Красная площадь
location = (59.9405, 30.3148)  # Эрмитаж

distance = geodesic(poi, location).km  # ~634 км
```
4. Кэширование результатов
```
cache = load_cache(cache_path)
if address in cache:
    return cache[address]  # быстрый возврат из кэша

location = geocode(address)
cache[address] = location  # сохраняем результат
save_cache(cache_path, cache)
```
5. Автоопределение колонки с адресами
```
def detect_address_column(fieldnames):
    lowered = {name.lower(): name for name in fieldnames}
    if "address" in lowered:
        return lowered["address"]
    return fieldnames[0]  # берем первую колонку
```

## 🐛 Обработка ошибок

Утилита корректно обрабатывает различные проблемные ситуации, выводя понятные сообщения:

| Ситуация | Статус | Что происходит |
|----------|--------|----------------|
| Пустой адрес | `empty` | Пропускается, записывается в output с пустыми координатами |
| Адрес не найден геокодером | `not_found` | Координаты пустые, расстояние не рассчитывается |
| Ошибка сети / таймаут | `error` | Запись в кэш как None, продолжение работы |
| Отсутствует input.csv | `SystemExit` | Скрипт завершается с сообщением об ошибке |
| Некорректный CSV | Зависит от структуры | Скрипт пытается определить формат, иначе берет первую колонку |
| Отсутствие интернета | `error` | Все запросы падают с ошибкой, но кэшированные работают |
| Превышение лимитов Nominatim | `error` | RateLimiter автоматически делает паузу между запросами |

## ⚙️ Коды возврата

Программа возвращает следующие exit codes:

| Код | Значение |
|-----|----------|
| 0 | Успешное выполнение (геокодирование завершено) |

## 🎯 Заключение
Geopy Mini-Project — это пример того, как с помощью geopy можно организовать промышленное геокодирование адресов в Python. Вместо того чтобы писать собственные обработчики API, управлять задержками и кэшированием вручную, проект использует готовый, проверенный и удобный инструмент.

Этот подход:

- Ускоряет разработку — не нужно разбираться с API каждого геокодера

- Обеспечивает надежность — RateLimiter защищает от блокировок

- Экономит ресурсы — кэширование исключает повторные запросы

- Дает качественные данные — расчет расстояний с учетом кривизны Земли

- Интегрируется в пайплайны — легко встраивается в ETL-процессы

Проект демонстрирует, что превращение "сырых" адресов в географические координаты может быть быстрым, экономичным и полностью автоматизированным.
